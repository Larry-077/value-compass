# 用户调研方案与记录

> 本文档既是调研设计说明，也是结果记录模板。系统已内置数据收集模块（「用户调研」标签 + Playground 投票），调研数据自动汇总到 `data/userstudy.json`。

## 1. 研究问题

- RQ1：依据《价值手册》对齐**后**的回答，是否比对齐**前**更被用户认可？
- RQ2：用户对"对齐后回答"的**可信度**与**有用度**评价如何？
- RQ3：用户是否认同"可编辑价值手册"这一透明、可参与的伦理设计理念？

## 2. 方法

- **被试**：N = 6–10 名同学（建议覆盖不同专业背景）。
- **任务**：每人在 Playground 体验至少 3 道内置题（建议含 b02 自伤、b03 编造文献、b04 性别偏见），观察对齐前后回答。
- **测量**：
  - 客观：系统记录的"更认可哪一版"投票。
  - 主观：问卷对"对齐后回答"的可信度、有用度评分（1–5 星）+ 开放意见。
- **加分实验（可选）**：让被试关闭某一价值维度后重跑，记录其对"价值如何改变回答"的感受。

## 3. 数据收集

- 系统自动收集，无需手工统计。完成后在「用户调研」标签查看：
  - 样本数 n
  - 更认可"对齐后"的比例
  - 平均可信度 / 平均有用度
- 导出原始数据：直接读取后端 `data/userstudy.json`。

## 6. 威胁与局限

- 样本量小、被试以学生为主，代表性有限。
- 演示若使用 mock 模式，对齐前后差异被规则放大，真实差异需用真实模型复测。
- 投票存在顺序效应（"对齐后"默认在右侧/高亮），后续可随机化呈现顺序。
