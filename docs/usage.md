# 系统使用说明 · ValueCompass 价值罗盘

## 0. 这个系统是什么、解决什么

ValueCompass 是一个**把"AI 价值对齐"做成可交互、可视化、可度量的系统**。它对应作业要求里的"系统实现"，主题落在 **AI 伦理 / 算法伦理 / 偏见 / 透明性 / 负责任 AI**。

它要回答并展示三件事：
1. **价值怎么编码**——把伦理价值写成一份有出处、带评分细则的《价值手册》（透明、可编辑）；
2. **价值怎么对齐模型**——用手册构造奖励信号，通过"自我批评-修订"和"Best-of-N"两种方式对齐，并能从人类偏好反过来学习价值权重（RLHF 核心）；
3. **怎么证明有效**——先构造一个能让模型"翻车"的困难数据集，再量化对齐前后的价值得分变化。

> 一句话：**它不是聊天机器人，而是一个"研究/教学用的价值对齐工作台"。**

## 1. 安装与启动

```bash
npm install
cp .env.example .env      # 填入模型令牌（见下）；不填则用内置 mock 离线演示
npm run dev               # 前端 5173 + 后端 8787
```
浏览器打开 http://localhost:5173 。右上角徽标显示当前用的是真实模型还是 mock。

**模型配置（.env）**——本系统把"生成器"和"评审"分开：
- `VC_GEN_*`：被对齐的"弱策略"（默认 智谱 `glm-4-flash`，国产、免费）
- `VC_JUDGE_*` / `VC_MODEL`：打分与批评的"强老师"（默认 `claude-sonnet-4-6`）

## 2. 七个页面分别在做什么（按 ①→⑦ 的流水线顺序）

| 页面 | 作用 | 产出 / 流向 |
|---|---|---|
| **① 价值手册** | 编辑价值原则：增删、调权重、启停；每条标注出处(UDHR/法规)，每维配评分细则(rubric) | 是所有页面的"价值标准"输入 |
| **② 困难数据集** | 用 4 类"加压技术"（越狱/编造/两难/偏见）把题目变难，跑生成器→评审→筛出"翻车"题 | 生成困难集，供 ③④⑥ 使用；产出各技术**翻车率**（贡献 C1） |
| **③ 对齐 Playground** | 单题看「弱模型原始 → 强评审批评 → 修订」三栏对比 + 价值评分雷达 | 可对结果投票 → 成为 ⑤ 的训练数据 |
| **④ Best-of-N** | 单题采样 N 个候选，用价值奖励选最优，画「价值分 vs N」曲线（推理期 RLHF） | 演示"用奖励对齐" |
| **⑤ Reward Model** | 在 ③④ 收集的人类偏好上训练 Bradley-Terry 奖励模型，对比"学到的权重 vs 手设权重" | 可把学到的权重写回 ① |
| **⑥ 评估面板** | 对所选题集（普通基准 / 困难集）批量跑对齐，汇总"对齐前 vs 后"整体提升 + 雷达 + 逐题表 | 出量化结论 |
| **⑦ 用户调研** | 汇总所有投票（更认可对齐前/后、可信度、有用度） | 加分项 |

**数据怎么流**：①②是"原料" → ③④⑥"用原料做对齐和评估" → ③④的投票汇入 ⑤"反过来改进①" → ⑦统计。

## 3. 推荐的端到端使用流程

1. **看价值标准**：进 ① 价值手册，浏览 5 个维度、13 条原则的出处与评分细则（可改权重做对照实验）。
2. **造难题**：进 ② 困难数据集，勾选 4 类技术 → 点"构建" → 得到困难集 + 各技术翻车率表。
3. **出核心对比（C1 + 对齐有效性）**：进 ⑥ 评估面板，
   - 先选「内置基准（普通题）」运行 → 记整体均分（通常很高）；
   - 再选「困难集」运行 → 整体均分明显更低，且"对齐前→后"提升明显。
   - 这一对比说明"普通题测不出问题、困难题才暴露不对齐，而对齐能修复"。
4. **看单题过程（演示/截图）**：进 ③ 选一道困难题跑，截三栏对比 + 雷达；进 ④ 跑同题看 N 曲线。
5. **训练奖励模型（E2/E3）**：在 ③/④ 多投几次票 → 进 ⑤ 训练 → 看人机一致性与学到的权重 → 写回 ①。
6. **用户调研**：找几位同学在 ③ 投票 → ⑦ 看汇总（加分）。

## 4. 常见问题

- **切换标签后，正在运行/已出的结果消失了？** 每个页面的结果是该页的局部状态，切走即清空。请在一个页面内完成查看/截图后再切换；评估这类长任务建议跑完再切。
- **全是满分 / 看不出差距？** 说明用的是普通题——现代模型已对齐。请在 ⑥ 选「困难集」，或在 ③④ 选困难题。
- **某些题报 403 / 余额不足？** 是模型供应商的限制（如 SiliconFlow 需充值）。换 `.env` 里的 provider（默认智谱 glm-4-flash 免费可用）。
- **没有 API 令牌也能用吗？** 能。不填 `.env` 即走内置 mock，界面与流程完整，仅内容为占位数据。
