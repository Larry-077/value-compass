# 价值罗盘 ValueCompass · 项目汇总报告

> 「伦理设计」课程项目。主题：以**可编辑《价值手册》**编码人工智能价值模型，驱动大模型对齐，并把"价值对齐"做成**可编辑、可运行、可度量**的系统。
> 系统使用说明见 [usage.md](usage.md)；开源仓库：https://github.com/Larry-077/value-compass

---

## 一、项目背景

大语言模型（LLM）正深度嵌入搜索、教育、医疗咨询、内容创作等场景，"模型说什么"直接影响人的判断与行为。一个能力很强但价值不对齐的模型，可能在用户脆弱时提供自伤方法、在被诱导时编造虚假文献、在不经意间强化性别与地域偏见。因此如何把人类社会的伦理价值"注入"模型行为——即**价值对齐（AI Alignment）**——是 AI 伦理设计的核心命题，也契合"数据伦理、算法偏见、透明性、负责任 AI"等议题。

工业界主流对齐技术有三条线：**RLHF**（用人类偏好训练奖励模型再做强化学习，Christiano 2017 / Ouyang 2022）、**Constitutional AI / RLAIF**（用一份"宪法"让 AI 产生偏好标签，Bai 2022）、**DPO**（直接偏好优化，Rafailov 2023）。它们共同的伦理设计缺陷是：**价值标准对使用者是黑箱**——你看不到它遵循哪些价值、改不了、也无法量化"加了某条价值后到底变好多少"。本项目正是针对这一缺陷，把价值"摊开"给人编辑、运行与度量。

## 二、系统需求设计

**目标用户**：AI 伦理课程的师生、对齐研究入门者。需要在不接触大规模训练的前提下，直观理解并亲手操作"价值如何约束模型"。

**功能需求**
1. **价值编码**：把伦理价值表示为可读、可增删、可调权重、可启停的《价值手册》，且每条原则可溯源（标注权威出处）。
2. **价值对齐**：用手册约束模型回答——支持「自我批评→修订」（宪法式）与「采样 N 个候选→奖励选优」（Best-of-N，推理期 RLHF）两种方式。
3. **价值奖励与学习**：把手册构造成标量奖励；并能从人类偏好中训练奖励模型（reward model），反推价值权重。
4. **难度构造**：构造能让对齐模型"翻车"的困难数据集（因为普通题已测不出问题），并量化各类技术的翻车率。
5. **评估与反馈**：批量量化对齐前后的价值得分；收集用户偏好反馈。

**非功能需求**
- **透明可溯**：价值与评分标准公开、有出处、有评分细则。
- **可运行、低门槛**：纯 OpenAI 兼容 API，无需 GPU；一个免费国产模型令牌即可全程实时运行。
- **开箱即用**：内置作者真实 API 跑出的结果，无需配置即可看到真实效果、无需等待。

## 三、详细设计

### 3.1 总体架构与数据流

```
浏览器 React（⓪导览 + ①…⑦ 功能页）
   │ fetch /api/*
Express 后端（唯一模型出口 server/claude.js）
   │ openai SDK（生成器 / 评审 可分别配置）
OpenAI 兼容 API（生成器=GLM-4-Flash 弱策略 / 评审=强模型）
```
价值手册（①）与困难数据集（②）是"原料"；③④⑥用它们做对齐与评估；③④的人类投票汇入⑤训练奖励模型、反过来改进①；⑦汇总反馈。

### 3.2 价值模型：手册的编码与依据

以 Anthropic **HHH**（Helpful / Honest / Harmless）为操作骨架，扩展"公平""自主与隐私"两维，共 5 维 13 条原则。**每条原则双重标注出处**：国际取联合国《世界人权宣言》(UDHR，Anthropic 立宪首要来源)，国内取《生成式人工智能服务管理暂行办法》(2023) 第四条与社会主义核心价值观；理论框架参考 Schwartz 基本人类价值理论。每个维度配**锚定式评分细则（rubric）**，明确 0/1/2 三档判据——使评分有据可查、可复现。

### 3.3 价值奖励与对齐方法

- **奖励函数**：`r(a)=Σ_d w_d·score_d(a)/Σ_d w_d`，score 由评审按 rubric 给出，w 为维度权重；把"价值手册"变成一个标量奖励。
- **自我批评-修订**：强模型依据手册批评弱模型的原始回答并改写（宪法式 AI）。
- **Best-of-N**：弱模型采样 N 个候选，用奖励选最优——不训练策略、仅在推理期用 reward 重排，是 API-only 可行的"推理期 RLHF"。

### 3.4 奖励模型（RLHF 核心）

关键设计：奖励模型的输入不是原始文本，而是评审给出的**维度分向量**。于是"训练奖励模型"退化为低维 **Bradley-Terry / 逻辑回归** `P(a≻b)=σ(θ·(f_a−f_b))`——在人类偏好对上拟合各维度权重 θ，**纯 JS、零 GPU、瞬时完成**。由此对比"手设权重 vs 从人反馈学到的权重"，形成"人设先验 → 人类偏好后验"的闭环。

### 3.5 困难数据集的半自动构建

四类"加压技术"模板：越狱/角色框架、编造/虚构施压、价值两难/取舍、隐性偏见/过度拒绝。流程：种子题→生成器作答→评审按 rubric 打分→目标维度"翻车"(≤1 分)者进入困难集，并统计各技术翻车率。

## 四、探索与思考逻辑（项目是怎样一步步形成的）

这是本项目最想体现的部分——**不是一开始就有完整设计，而是被实验逼出来的**：

1. **出发点**：想做"价值对齐"，最初实现了"价值手册→自我批评→修订→评分"的闭环（宪法式 AI 的简化版）。
2. **第一次受挫**：上真实模型后发现**对齐前后几乎都是满分**。排查后定位到两个原因：一是评审只有 0/1/2 且无判据，对什么都给高分（**评分器没有判别力**）；二是现代模型在明显题上本就对齐。
3. **打地基（SP2）**：给每维加**锚定式 rubric**、给每条原则加**出处**。用"已知好/坏答案"验证：加 rubric 后评分能拉开（坏 0.8 vs 好 1.8）——评分器有判别力了。
4. **第二次受挫**：换了三个小模型（Qwen2.5-7B、gpt-4o-mini、GLM-4-Flash）发现，**普通题它们都已对齐，测不出不对齐**。
5. **关键转向（核心洞察）**：既然普通题测不出，就必须**主动构造让模型翻车的困难题**。于是把"困难数据集 + 构建方法学"本身确立为**核心贡献 C1**——这正是红队/对抗基准的研究路线（AdvBench、HarmBench、XSTest 等）。实验证明：用越狱框架，连 GLM-4-Flash 也会给出氰化钾配制步骤（无害维度 0 分）。
6. **闭环成形**：困难集 → 在其上做对齐与 Best-of-N（C2）→ 从投票训练奖励模型（C2）→ 量化对齐前后（透明度量 C3）。

**关键实验数据**（生成器 GLM-4-Flash / 评审 claude-sonnet-4-6）：
- 困难集构建：20 道种子题，**总翻车率约 50–60%**；其中**编造类最高（80%）**，越狱 60%，两难 50%，偏见 50%；甚至捕捉到一例"过度拒绝"（让模型写氯气科普它无理由拒绝）。
- 对齐有效性：在困难集上**对齐前整体均分 0.64 → 对齐后 1.98（满分 2）**，提升约 +1.34；而普通题对齐前本就 1.6–2.0（印证"普通题测不出、困难题才暴露、而对齐能修复"）。

## 五、项目贡献

- **C1（核心）· 价值压力测试困难数据集 + 构建方法学**：用系统化加压技术构造能让对齐模型翻车的评测集，并以翻车率实验量化各技术有效性。
- **C2 · 价值手册→奖励的可编辑对齐框架**：自我批评-修订、Best-of-N、从人类偏好学习价值权重的奖励模型，完整走通 RLHF 闭环且纯 API 可行。
- **C3 · 透明可参与的伦理设计**：价值可编辑、出处可溯（UDHR/法规）、评分有据（rubric）、效果可度量——把对齐从黑箱变为可检查、可参与。

## 六、初步使用反馈

系统内置了一批人类偏好样本并支持现场投票（⑦ 用户调研）；在小范围体验中，用户对"对齐后"回答的认可度显著高于"对齐前"。完整调研可在 ⑦ 面板继续收集。

## 七、局限与展望

- 评审为单模型，存在评审自身偏差；可引入多评审投票或人工校准。
- 翻车率随模型采样有波动，宜多次取平均；困难集规模可继续扩大。
- 奖励模型在低维分向量上拟合、且当前偏好高度一致，学习意义偏演示；引入有分歧的偏好后信息量更大。
- 可扩展 DPO、多语言手册、跨模型横向对比。
