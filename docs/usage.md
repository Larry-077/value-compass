# 系统使用说明 · ValueCompass 价值罗盘

## 0. 这是什么

一个把"AI 价值对齐"做成**可交互、可视化、可度量**的系统（不是聊天机器人）。你写一份《价值手册》编码伦理价值，系统用它约束模型、给回答打分、并从人类反馈学习价值权重。项目背景/设计/贡献见 [report.md](report.md)。

## 1. 启动

```bash
npm install
npm run dev        # 前端 5173 + 后端 8787
```
打开 **http://localhost:5173**。

- **无需任何 API Key**：每页默认显示作者用真实模型跑出的结果（内置于 `src/data/demo.js`），打开即见、无需等待。
- **想亲自实时跑**：`cp .env.example .env`，在 `.env` 填一个**免费智谱 GLM 令牌**（[open.bigmodel.cn](https://open.bigmodel.cn/)），即可全站实时调用（生成+评审都用这一个 key）。
- 端口被占报 `EADDRINUSE`：`lsof -ti tcp:8787 tcp:5173 | xargs kill -9` 后重试。

## 2. 七个页面在做什么（按 ①→⑦ 流水线）

| 页面 | 作用 |
|---|---|
| **⓪ 项目导览** | 一页讲清"这是什么、探索思考过程、三项贡献"，建议先看 |
| **① 价值手册** | 5 维 13 条价值原则：可增删/调权重/启停；每条标注出处(UDHR/法规)，每维有评分细则 |
| **② 困难数据集** | 用 4 类加压技术构造让模型"翻车"的题，看各技术翻车率（结果自动保存，默认加载上次） |
| **③ 对齐 Playground** | 单题看「对齐前→批评→对齐后」三栏 + 评分雷达；可投票 |
| **④ Best-of-N** | 采样 N 个候选，用价值奖励选最优，看"价值分 vs N"曲线 |
| **⑤ Reward Model** | 在人类偏好上训练奖励模型，对比"学到的权重 vs 手设权重"，可写回手册 |
| **⑥ 评估面板** | 对所选题集批量量化"对齐前 vs 后"整体得分 + 雷达 |
| **⑦ 用户调研** | 汇总投票统计 |

> 每个功能页顶部都有「引导头」：📍这步做什么 / 👉你要做的 / 🎯体现哪个贡献 / 下一步按钮。

## 3. 推荐体验路线

1. **⓪ 导览**读一遍 → **① 手册**看价值如何编码。
2. **② 困难数据集**：默认已显示作者跑好的翻车率（编造 80%、越狱 60% 等）；想自己跑就点"构建"。
3. **⑥ 评估面板**：分别选「内置基准(普通题)」与「困难集」——普通题均分高、困难题低且对齐后明显提升，这是核心结论。
4. **③/④**：快速选题里有「⚡示例（秒出）」直接看真实结果；想试自己的句子，在文本框输入再点运行（实时，约 30 秒）。
5. **⑤ Reward Model**：看面板内"怎么读"折叠说明；点训练即在内置偏好上拟合权重。

## 4. 现场试自己的问题

去 **③ 对齐 Playground**：上方下拉可选预置题（含秒出示例），也可在文本框**自由输入一句话**，点"运行对齐流水线"，即现场生成「对齐前 vs 对齐后」两栏对比（需在 `.env` 配 GLM 令牌；否则用内置示例）。

## 5. 常见疑问

- **为什么有些回答是"未经价值审校的回答"占位文本？** 没配 key 时实时运行会回退到占位数据；配上 GLM 令牌即为真实结果。
- **Reward Model 为什么不调 API 也能训练？** 训练输入是评分阶段已算好的维度分向量，拟合权重是纯数学，瞬时完成。详见该页"怎么读"折叠说明。
